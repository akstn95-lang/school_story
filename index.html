도움말

앱 사용법과 설계 원칙을 안내합니다.

이 앱의 구조 v2.4

① 종합 진단(전체 방향성) → ② 규정 검사(적발) → ③ 정성 채점(평가) → ④ 나침반 제안(고쳐쓰기 방향) → ⑤ AI 연계 병합(초안 생성). 어떤 단계도 교사의 최종 확인을 대체하지 않습니다. AI 병합 기능도 결과를 자동 반영하지 않고 ‘적용’을 눌러야만 원문에 반영됩니다.

설계 원칙 (요약)

1) 점수는 서술 완성도 지표이며 합격 예측이 아님  2) 사례 원문은 노출하지 않고 패턴만 제공  3) 최종 입력 권한과 책임은 교사에게 있음  4) 업로드한 파일과 API 키는 브라우저 밖으로 전송되지 않음(파일은 로컬 처리, API 키는 Anthropic 서버 호출에만 사용)  5) 입력한 초안·이력은 ‘기록 초기화’를 누르기 전까지 이 브라우저에 남아 있음

버전 기록

v2.4 — 챗GPT·제미나이 기본 모델명 지정
AI 서비스를 챗GPT로 선택하면 기본 모델명이 gpt-5.4-mini로, 제미나이는 gemini-2.0-flash로 채워지도록 설정함(직접 다른 모델명으로 바꿀 수 있음).
v2.3 — 검토 이력에 학번·이름·진로 희망분야 함께 저장
종합 진단 탭에 학생 정보(학번·이름) 입력창을 추가함. 파일을 불러오면 인쇄 꼬리말에서 이름을 자동으로 찾아 채우며, 직접 고칠 수 있음. 분석할 때마다 검토 이력에 학번·이름과 함께, 그 항목의 학년에 해당하는 진로 희망분야를 자동으로 매칭해 함께 저장함. 텍스트·Word 파일로 내보낼 때도 이 정보가 포함됨.
v2.2 — API 키 저장(30일 자동 삭제), 검토 이력 수정·삭제, 세특 워터마크 제거 개선
API 키를 서비스별로 이 브라우저에 저장해 다시 입력할 필요가 없게 하고, 저장한 지 30일이 지나면 자동으로 삭제되도록 함(직접 삭제하는 버튼도 추가). 검토 이력 탭에서 항목별로 ‘수정’을 누르면 원문을 바로 고쳐 저장할 수 있고(저장 시 규정검사·점수를 다시 계산하고, 분석 작업의 해당 슬롯에도 함께 반영됨), ‘삭제’로 개별 항목을 지울 수 있게 함. 세특을 불러올 때 인쇄 워터마크(학교명/날짜시간/IP/이름)가 날짜와 시간 사이에 공백이 없는 형식이면 걸러지지 않던 문제를 수정함.
v2.1 — 새 항목 선택창 한 줄 배치, AI 서비스 선택(Claude·챗GPT·제미나이), 검토 이력 선택 저장
'새로 작성할 내용'의 학년·학기·기재영역·과목명·전형유형·지원계열 6개 선택창이 한 줄에 모두 들어가도록 폭을 조정함. AI 연계 병합에서 Claude 외에 챗GPT(OpenAI)·제미나이(Google) 중에서 서비스를 선택할 수 있게 하고, 서비스를 바꾸면 모델명과 API 키 안내가 자동으로 바뀌도록 함(챗GPT·제미나이는 브라우저 직접 호출이 막혀 있으면 규칙 기반으로 자동 전환됨). 검토 이력 탭에 항목별 체크박스와 전체 선택/해제, 선택한 항목만 텍스트 파일 또는 Word·구글독스에서 열리는 파일로 저장하는 기능을 추가하고, 더 이상 쓰지 않는 리포트 인쇄 버튼은 제거함.
v2.0 — 분석 작업 레이아웃 재배치: 원문 입력 폭 확대
3등분이던 원문입력·규정검사·점수나침반 배치를 2단으로 바꿈. 규정 준수 검사를 점수·나침반 위쪽으로 옮겨 오른쪽 한 열에 세로로 쌓고, 그만큼 확보된 공간으로 원문 입력창의 폭을 넓힘.
v1.9 — 지난 생기부 내용 확인창을 전체 폭 · 세로 스크롤 없는 레이아웃으로 변경
'지난 생기부 내용 확인'의 개별 기록 카드가 여러 열로 나란히 배치되던 방식에서, 카드 하나하나가 창 전체 폭을 쓰고 아래로 쌓이는 방식으로 바꿈. 카드 내부의 세로 스크롤을 없애 전체 내용이 잘리지 않고 바로 보이도록 함(기록이 많을 때는 바깥 창 전체가 스크롤됨).
v1.8 — 종합 진단에 진로 희망분야 자동 검색 카드 추가
종합 진단 탭의 '현재 학년·학기' 바로 아래에 진로 희망분야 카드를 추가함. 불러온 생기부의 진로활동 기록에서 '희망분야' 표기를 학년별로 자동으로 찾아 보여줌. 페이지 병합 등으로 줄바꿈이 사라진 경우에도 다음 문장이 시작되는 지점에서 정확히 끊어 표시하도록 처리함.
v1.7 — AI 초안 작성 시 취약 공통요소를 집중 보완하도록 설정
AI 연계 병합 시, 기존 문구와 추가 자료를 합쳐 놓고 아직 충족되지 않은 공통요소를 찾아(전체 진단이 있으면 그중에서도 전체적으로 취약한 순서를 우선) 최대 3개까지 골라 프롬프트에 '집중 보완 요소'로 전달함. 관련 소재가 있으면 그 요소가 잘 드러나도록 비중 있게 서술하고, 소재가 전혀 없으면 억지로 끼워 넣지 않도록 규칙을 명시함. 결과 화면에 어떤 요소를 집중 보완하려 했는지, 실제로 충족되었는지(✓)를 함께 표시함.
v1.6 — 두 선택창의 상호 간섭 제거, AI 병합 글자수 계산 오류 수정
새로 작성할 내용의 학년을 바꾸면 지난 생기부 확인창의 학년도 따라 바뀌던 문제를 고쳐 두 선택창이 완전히 독립적으로 동작하도록 함. AI 연계 병합에서 목표 글자수를 '한도 − 기존 글자수'로 계산해, 기존 문구가 이미 한도를 넘는 경우 목표가 지나치게 작아져(심하면 40자) 한두 문장짜리 부실한 결과가 나오던 문제를 수정함 — 이제 목표는 항상 영역 전체 한도이며, 기존 문구가 한도를 넘는 경우에도 사실을 유지한 채 표현을 압축해 한도를 지키도록 안내함.
v1.5 — 지난 생기부 확인창을 화면 상단·전체 폭으로 분리, 독립 선택 가능
'지난 생기부 확인' 창을 원문 입력창 안에서 분리해 화면 맨 위, 화면 전체 폭을 쓰는 별도 영역으로 옮김. 지난 기록을 학년·학기·기재 영역·과목명으로 직접 선택해 확인할 수 있게 하고(과목명을 비우면 해당 영역 전체 확인), 그 아래에 새로 작성할 내용의 학년·학기·기재 영역·과목명·전형 유형·지원 계열 선택창을 별도 줄로 배치함. AI 연계 병합은 지난 기록 확인창의 선택과 무관하게, 지금 새로 작성 중인 항목을 기준으로 관련 기록을 자동으로 찾아 참고함.
v1.4 — 지난 생기부 내용 창을 항상 보이는 읽기 전용 창으로 개선
‘지난 생기부 내용’을 클릭해야 펼쳐지던 방식에서, 관련 기록이 있으면 바로 보이는 별도의 읽기 전용 창으로 바꿈. 원문 입력창과 시각적으로 분명히 구분되도록 배경·테두리를 다르게 주고 ‘여기서는 수정할 수 없다’는 안내를 명시함.
v1.3 — 종합 진단에 공통요소 설명 추가, 지난 기록을 참고한 AI 이어쓰기
‘종합 진단’ 탭의 취약 공통요소 목록에 각 요소가 무엇을 뜻하는지 설명을 함께 표시함(예시 문장은 규정·평가 사전 탭에서 확인). ‘분석 작업’ 탭에 지금 쓰려는 항목(세특은 같은 과목)과 관련된 지난 학기 기록을 보여주는 참고 패널을 추가함. AI 연계 병합 시 지난 기록과 자연스럽게 이어지는 지점이 있을 때만 가볍게 연결하고, 관련성이 없으면 언급하지 않도록 프롬프트를 조정함.
v1.2 — 학년/학기 작성 단위 재정비, 기록 유지, 대시보드 정리
자율·동아리·진로·행동특성(및 개인별 세특)은 학기 구분 없이 학년 전체가 하나의 이어지는 초안이 되도록 바꿈. 1~2학년 공통과목 7개 세특도 학기와 무관하게 하나의 초안(500자·1500바이트)으로 관리되도록 함. 항목(학년·학기·영역·과목)을 선택하면 기존에 저장한 초안을 자동으로 불러와 이어 쓸 수 있게 함. 입력한 초안·이력·불러온 파일이 ‘기록 초기화’를 누르기 전까지 새로고침해도 남아 있도록 저장 기능을 추가함. 역할이 모호했던 대시보드 탭을 없애고, 12대 공통요소 설명은 예시 문장과 함께 규정·평가 사전 탭으로 옮김. 분석 작업 탭에서 불필요한 학생 코드 입력란을 없앰. 리로스쿨·챗봇 등 자주 쓰는 사이트로 바로가기(기본 숨김) 기능과 작성한 초안 복사 버튼을 추가함.

자주 묻는 질문

글자수와 바이트는 왜 둘 다 보여주나요?
화면의 글자수는 한글/워드프로세서에서 흔히 쓰는 방식대로 띄어쓰기(공백)를 제외하고 셉니다. 반면 나이스(NEIS)의 실제 입력 한도는 Byte 단위이며 한글 1자를 3Byte, 공백을 1Byte로 계산해 띄어쓰기도 포함됩니다. 두 지표가 다른 기준이라 값이 다르게 보일 수 있어 함께 표시합니다 — 실제 나이스 입력 가능 여부는 바이트 값을 기준으로 확인해 주세요.
왜 자동으로 문장을 고쳐주지 않나요?
기재요령은 “생성형 AI가 만든 자료를 그대로 서술형 항목에 입력하는 행위”를 신뢰도 저해 사례로 명시하고 있습니다. 이 앱은 실제 관찰되지 않은 사실을 창작하지 않도록, 진단과 질문형 제안까지만 하고 최종 문장은 교사가 작성하도록 설계했습니다.
세 자료 중 무엇이 가장 중요한가요?
우선순위가 아니라 역할이 다릅니다. 기재요령을 어기면 아무리 잘 쓴 문장도 무효이므로 최우선 필터이고, 동국대 가이드북은 “잘 쓴 문장”의 기준을, 사례집은 “어떻게 그 기준에 도달했는지”의 실증 패턴을 제공합니다.
왜 독서활동상황과 봉사활동 특기사항은 다루지 않나요?
독서활동상황은 2024학년도 대입부터 상급학교 진학자료로 제공되지 않아(대입 미반영) 이번 버전에서는 코칭 대상에서 제외했습니다. 봉사활동은 2022 개정 교육과정부터 별도의 정성적 특기사항 없이 객관적 실적(50자)만 입력하도록 되어 있어, 마찬가지로 이번 버전의 범위에서 제외했습니다. 두 항목 모두 ‘규정·평가 사전 > 영역별 최대 글자수’ 탭 하단에서 제외 사유를 확인할 수 있습니다.
글자수 한도가 왜 "학년 단위"로 계산되나요?
2026학년도 기재요령은 과목별 세부능력 및 특기사항을 제외한 모든 서술형 항목(자율·자치활동, 동아리활동, 진로활동, 행동특성 및 종합의견, 개인별 세특)의 최대 글자수를 학기가 아닌 학년 단위로 규정합니다. 이 앱에서는 이 항목들을 아예 학기 구분 없이 하나의 이어지는 초안으로 관리합니다 — 학년·영역(·과목)을 선택하면 지금까지 저장한 내용을 자동으로 불러오므로, 학기가 바뀌어도 같은 초안을 계속 이어서 고치면 됩니다.
공통과목 세특의 글자수 규칙은 무엇이 다른가요?
2022 개정 교육과정을 적용받는 1~2학년의 공통과목 7개(공통국어·공통수학·공통영어·통합사회·통합과학·한국사·과학탐구실험)는 1학기와 2학기의 세특을 합산해 500자 이내로 써야 합니다. 이 앱에서는 공통과목명을 입력하면 학기 구분 없이 하나의 초안(500자·1500바이트)으로 관리됩니다. 그 외 일반 선택과목은 과목당·학기당 각각 독립된 500자가 주어지며, 학기가 지나면 원칙적으로 다시 손대지 않습니다.
파일을 업로드하면 서버로 전송되나요?
아니요. PDF·엑셀·TXT 파일은 브라우저(자바스크립트) 안에서만 열리고 분석됩니다. 어떤 서버에도 파일이 업로드되지 않습니다. 다만 영역 인식은 제목 문구를 기준으로 한 자동 추정이므로, 불러온 뒤 영역·과목·학년이 올바른지 반드시 확인해 주세요.
AI 연계 병합 기능은 어떻게 동작하나요?
‘추가 자료’ 입력창에 새로 관찰한 메모를 적고 병합 버튼을 누르면, 기존 원문 입력창의 내용과 합쳐 하나의 문장으로 다시 씁니다. Anthropic API 키를 입력하면 Claude API가 실제로 문장을 생성하고, 키가 없으면 문장을 단순히 이어붙이는 규칙 기반 방식으로 대체됩니다. 어느 쪽이든 결과는 미리보기로만 표시되고, ‘이 초안을 원문에 적용’을 눌러야 실제 원문 입력창에 반영됩니다.
API 키는 안전한가요?
입력한 키는 이 브라우저 탭의 메모리에만 잠깐 보관되며 저장소에 기록되지 않고, 선택한 서비스(Claude·챗GPT·제미나이) 호출 외의 용도로 쓰이지 않습니다. 다만 이 방식은 개인 테스트·소규모 사용에 적합하며, 여러 교사가 함께 쓰는 정식 서비스로 확장할 때는 GAS 백엔드 등 서버 측에서 키를 관리하도록 구조를 바꾸는 것을 권장합니다.
챗GPT·제미나이도 잘 되나요?
세 서비스 모두 공식 API 형식에 맞춰 연결해 두었습니다. 다만 챗GPT·제미나이는 브라우저에서 직접 호출하는 것을 막아둔 경우가 있어(서비스 정책은 바뀔 수 있음), 실제로 호출이 안 되면 오류 메시지와 함께 규칙 기반 병합으로 자동 전환됩니다. 안정적으로 쓰려면 Claude를 권장하며, 다른 서비스를 쓰고 싶다면 서버(GAS 등)를 거쳐 호출하는 구조로 바꾸는 것이 좋습니다.
입력한 내용이 새로고침해도 남아 있나요?
이 파일을 다운로드해서 직접 열거나 별도 서버·GAS 등에 올려 사용하는 경우에는 브라우저 저장소에 남아 있어 새로고침해도 유지되며, ‘종합 진단’ 탭의 ‘🗑 기록 초기화’ 버튼을 눌러야만 지워집니다. 다만 claude.ai 채팅 안의 미리보기 화면처럼 저장소 접근이 막힌 환경에서는 저장이 되지 않을 수 있고, 그 경우 세션 동안은 정상 동작하되 새로고침하면 초기화됩니다.
항목을 다시 선택했더니 입력창 내용이 바뀌었어요.
의도된 동작입니다. 학년·학기·영역·과목을 바꾸면 그 항목에 이미 저장된 초안이 있는지 확인해서, 있으면 자동으로 불러오고 없으면 빈 칸으로 둡니다. 지금 입력 중인 내용을 남기고 싶다면 선택을 바꾸기 전에 먼저 ‘분석하기’를 눌러 저장해 주세요.
